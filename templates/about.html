О проекте

Вариант 4 — Классификация блюд по фотографии

Цель

Разработать систему компьютерного зрения для определения категории блюда по изображению с выводом top-3 наиболее вероятных классов, сравнением 5 архитектур CNN и демонстрационным веб-модулем.

Данные

ДатасетFood-101 (Bossard et al.) — подмножество 20 классов
Тип разметкиКлассовая (одна метка на изображение)
Объём20 000 изображений (1000 на класс в полном Food-101)
Разбиение70% train / 15% validation / 15% test
ОграниченияТолько исследовательское использование; не для медицинских/коммерческих целей без дообучения

Архитектуры (5 разных семейств)

  1. ResNet-18 — остаточные связи, глубина 18 слоёв
  2. ResNet-50 — углублённая residual-сеть
  3. MobileNetV3-Small — лёгкая мобильная архитектура
  4. EfficientNet-B0 — compound scaling (лучшая модель)
  5. DenseNet-121 — плотные связи между слоями

Метрики

Accuracy, Precision, Recall, F1-score, Confusion Matrix — см. разделы «Архитектуры» и «Confusion Matrix».

Технологии

Python 3, Flask, PyTorch, torchvision, SQLite, openpyxl, reportlab

Запуск

pip install -r requirements.txt
python run.py

Откройте http://127.0.0.1:5000
